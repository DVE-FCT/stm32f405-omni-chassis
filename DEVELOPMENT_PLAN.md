### 2.3 3 级 PID 定义

```
位置环 PID (外层, 50Hz 执行)
  输入: 目标位姿 (x_target, y_target, theta_target)
  反馈: 融合后的当前位姿 (x, y, theta)
  输出: 目标速度 (vx_target, vy_target, w_target)
  Kp ≈ 1.0 ~ 2.0

速度环 PID (中层, 200Hz 执行)
  输入: 目标速度 (vx, vy, w)
  反馈: 编码器测得的实际轮速转换的速度
  输出: PWM 占空比变更量
  Kp ≈ 0.5 ~ 1.0

占空比输出 (内层)
  直接输出 PWM 值 + 方向控制信号
  限幅: 0 ~ ARR (8399 或 4199)
```

### 2.4 通信协议

#### LoRa (USART3) 指令格式

```
PC → STM32:
{"cmd":"move","x":100.0,"y":0.0,"theta":0.0}   // 移动到绝对坐标 (cm)
{"cmd":"speed","vx":0.0,"vy":0.0,"w":0.0}      // 开环速度控制
{"cmd":"stop"}                                    // 急停
{"cmd":"reset"}                                   // 复位里程计

STM32 → PC (自动上报, 20Hz):
{"x":10.5,"y":20.3,"theta":45.2,"vx":0.1,"vy":0.0,"w":0.05}
```

## 三、分阶段开发计划

### 阶段 1: 电机单路驱动测试 (基础验证)

**目标**: 确认 1 个电机能正反转、调速

**CubeMX**: 无需改动

**新增文件**: `motor.c/h`

**代码内容**:
- `Motor_Init()` — 初始化所有电机
- `Motor_SetSpeed(motor_id, speed)` — speed 范围 -100 ~ +100
- 内部: 根据 speed 符号设置方向 GPIO + 绝对值写 PWM CCR

**测试方法**:
- main.c 中依次让 M1 正转 2s → 停止 1s → 反转 2s
- 示波器/万用表观察 PA8 有 PWM 波形变化
- PC0/PC1 方向引脚电平变化

**成功标志**: ✅ 电机正反转正常，PWM 可调

---

### 阶段 2: 编码器读取 + 轮速计算

**目标**: 读取编码器脉冲，换算为轮速 (rpm)

**新增文件**: `encoder.c/h`

**代码内容**:
- `Encoder_Init()` — 启动 4 路编码器定时器
- `Encoder_GetCount(motor_id)` — 读取当前计数值
- `Encoder_GetSpeed(motor_id)` — 计算转速 (脉冲差/dt)
- 定时 10ms 调用 `Encoder_Update()` 计算速度

**测试方法**:
- M1 用手转动, printf 输出 TIM3->CNT 变化
- moto_set pwm + printf 输出 4 路 rpm

**成功标志**: ✅ 4 路编码器读数正确，rpm 数值合理

---

### 阶段 3: 陀螺仪接入

**目标**: 维特陀螺仪通过 USART2 接收数据，拿到角度

**复用文件**: `wit_gyro/` (从 f405_test 项目复制)

**操作**:
- 复制 `wit_c_sdk.c/h`, `wit_gyro.c/h`, `wit_instance.cpp` 到 Core/Src
- 在 USART2 RX 回调中: `WitSerialDataIn(byte)`
- 确保陀螺仪波特率匹配（维特默认 9600 或 115200）

**测试方法**:
- printf 输出 `fAngle[2]` (Yaw)、`fAngle[0]` (Roll)、`fAngle[1]` (Pitch)
- 用手旋转陀螺仪，观察角度变化

**成功标志**: ✅ Yaw/Roll/Pitch 数据实时更新，角度准确

---

### 阶段 4: 里程计积分

**目标**: 从 4 路轮速 + IMU 角度，算出底盘位姿 (x, y, θ)

**新增文件**: `odometry.c/h`

**公式** (麦轮底盘):
```
vx = (v1 + v2 + v3 + v4) * R / 4
vy = (-v1 + v2 - v3 + v4) * R / 4
w  = (-v1 + v2 + v3 - v4) * R / (4 * (Lx + Ly))

位置积分 (世界坐标系):
x += (vx * cos(θ) - vy * sin(θ)) * dt
y += (vx * sin(θ) + vy * cos(θ)) * dt
θ += w * dt
```

**测试方法**:
- 手动推小车走直线 1m, 串口输出里程计 (x,y)
- 手动旋转小车 90°, 串口输出 θ 变化

**成功标志**: ✅ 位移和角度趋势正确（允许纯编码器漂移误差）

---

### 阶段 5: 互补滤波融合

**目标**: 编码器里程计 + IMU 融合，输出更稳定的 (x, y, θ)

**新增文件**: `fusion.c/h`

**融合策略**:
```
θ_fused = 0.98 * (θ_fused + w_gyro * dt) + 0.02 * θ_encoder
x_fused = x_encoder      // 编码器 x 直接使用(IMU 无位置)
y_fused = y_encoder      // 编码器 y 直接使用

或者简化为:
θ 来源: IMU 陀螺仪积分 (短期精度高)
x,y 来源: 编码器轮速积分
定期用 θ 修正 x,y 的方向分量
```

**测试方法**:
- 推车走 S 形路径, printf 对比纯编码器 vs 融合后的轨迹
- 快速自转，观察融合 θ 是否比纯编码器 θ 更稳定

**成功标志**: ✅ 融合后的 θ 漂移明显低于纯编码器

---

### 阶段 6: PID 控制器

**目标**: 实现 3 级串联 PID，能精确控制位置

**新增文件**: `pid.c/h`

**测试分两步**:

**6a. 速度环测试**:
- 给定目标速度 (vx, vy, w)
- PID 调节 PWM 使实际速度跟踪目标
- 串口指令: `{"cmd":"speed","vx":10.0,"vy":0.0,"w":0.0}`
- 用手轻轻阻挡车轮 → 自动加速对抗

**6b. 位置环测试**:
- 给定目标位置 (x, y, θ)
- 外层 PID 计算目标速度 → 中层 PID 跟踪
- 串口指令: `{"cmd":"move","x":50.0,"y":0.0,"theta":0.0}`
- 小车从原点出发 50cm 后停止

**成功标志**: 
- ✅ 速度环: 车轮转速能快速收敛到目标值
- ✅ 位置环: 小车移动到目标位置，误差 < 2cm

### 阶段 7: 静止自复位

**目标**: 静止时被推走，自动回到原位

**实现**:
```
if (位置模式 && 目标速度为0) {
    if (|Δx| > 2cm || |Δy| > 2cm || |Δθ| > 3°) {
        位置环 PID 自动产生反向速度 → 驱动回原位
    }
}
```

**测试方法**:
- 小车停在原点，用手推走 30cm
- 松手 → 小车自动返回原点

**成功标志**: ✅ 推走后 2 秒内开始回归，最终误差 < 3cm

---

### 阶段 8: LoRa 通信协议

**目标**: 通过 LoRa (USART3) 接收指令、上报位姿

**新增文件**: `protocol.c/h`

**功能**:
- 接收 JSON 解析: `{"cmd":"move","x":100.0,"y":50.0,"theta":0.0}`
- 定期上报: `{"x":...,"y":...,"theta":...,"vx":...,"vy":...,"w":...}`
- 支持命令: move, speed, stop, reset

**测试方法**:
- PC 端串口助手发送 JSON 指令
- 小车执行并回传位姿

**成功标志**: ✅ 指令解析正确，遥测数据准确

## 五、关键公式速查

```
麦轮运动学模型 (R=轮半径, Lx=轮距X半长, Ly=轮距Y半长):
  vx =  ( v1 + v2 + v3 + v4) * R / 4
  vy = (-v1 + v2 - v3 + v4) * R / 4
  w  = (-v1 + v2 + v3 - v4) * R / (4 * (Lx + Ly))

  逆解 (目标速度 → 各轮转速):
  v1 = (vx - vy - w*(Lx+Ly)) / R
  v2 = (vx + vy + w*(Lx+Ly)) / R
  v3 = (vx - vy + w*(Lx+Ly)) / R
  v4 = (vx + vy - w*(Lx+Ly)) / R

里程计 (世界坐标系):
  x += (vx * cos(θ) - vy * sin(θ)) * dt
  y += (vx * sin(θ) + vy * cos(θ)) * dt
  θ += w * dt

互补滤波:
  θ = α * (θ + w_gyro * dt) + (1-α) * θ_encoder
  推荐 α = 0.98
```
